# 剧情树编辑器

## 概述

`editor/` 目录下提供了一个可视化的剧情树节点编辑器（基于 Vue 3），用于在浏览器中直接编辑和调试游戏剧情。

## 访问方式

在浏览器中打开 `editor/editor.html` 即可使用编辑器。编辑器与主应用共用数据源（`game/` 目录的 JS 模块）。

## 节点系统

编辑器将可视化画布上的所有元素统一为 **三种节点类型**，地位完全等同：

### 节点类型

| 类型 | 标识 | 说明 |
|------|------|------|
| **章节节点** (chapter) | `chapter_xxx` | 包含游戏步骤（对话/分支/跳转/结局触发），是剧情的主要载体 |
| **结局节点** (ending) | `_end_xxx` | 映射到游戏结局列表 (`gameEndings`)，显示结局标题和描述 |
| **入口节点** (entry) | 任一章节节点 | 被标记为 `entryPoint` 的章节节点，游戏由此开始执行 |

### 节点交互（三种节点地位等同）

- **拖拽** — 所有节点可在画布上自由拖拽定位
- **选中** — 单击选中，显示对应编辑面板
- **框选** — 按住左键拖拽框选多个节点
- **右键菜单** — 显示类型相关的操作菜单
- **删除** — 支持单选/多选删除
- **分组** — 任意节点均可加入/移出分组
- **缩放/平移** — 所有节点参与画布变换

### 节点样式

- **章节节点**: 默认暗色主题，左侧色条区分根节点（蓝色）、叶节点（黄色）
- **结局节点**: 红色边框 + 圆角较大，标记 🎬
- **入口节点**: 绿色边框 + 发光效果，标记 🚪

### 技术实现

节点系统在 `editor-app.js` 中通过以下机制实现：

- **`getNodeType(id)`** — 判定节点类型（`'chapter'` / `'ending'`）
- **`treeNodes` 计算属性** — 统一构建三种节点数据，包含位置、端口、样式
- **`selectNode(id)`** — 统一选中逻辑，设置 `selectedChapterId` 或 `selectedEndingId`
- **`deleteChapter(batchIds?`)** — 统一删除逻辑，根据节点类型执行不同清理

## 功能

### 可视化剧情树

以 **SVG 连线 + 可拖拽节点** 的方式展示章节之间的跳转关系：

- **节点** = 章节 / 结局（显示名称、步骤数、出入口数）
- **连线** = 章节跳转（`jumpChapter` 指向关系，或结局触发）
- **端口** = 底部圆形出口 / 顶部方形入口（可拖拽重连）

编辑器会自动分析章节间的跳转依赖，计算入度/出度，识别根节点和叶节点。

### 章节 CRUD

- 添加新章节 (`Ctrl+N`)
- 删除现有章节 (`Delete`)
- 重命名章节
- 复制章节 (`Ctrl+D`)

### 结局节点管理

- **添加结局节点** — 右键画布空白 → 「在此新建结局」或 `Ctrl+Shift+N`
- **编辑结局** — 双击/右键结局节点 → 打开结局编辑面板
- **删除结局** — 选中后按 `Delete` 或右键 → 「删除节点」
- 结局编辑面板中可修改标题、描述，查看来源章节

### 步骤编辑

每个步骤是一个可编辑的表单项，支持所有步骤类型：

**对话步骤字段：**
- `sceneId` — 场景选择（下拉菜单）
- `characterId` — 角色选择
- `text` — 对话文本
- `effects` — 效果列表（多选）
- `screenEffect` — 粒子特效（选择器）
- `cgChanges` — CG 变化（动作 + CG ID + 动画）
- `gainItem` / `loseItem` — 物品变更
- `flag` — Flag 设置
- `jumpChapter` — 跳转目标

**分支步骤字段：**
- 选项列表编辑（增删改）
- 每个选项的 `text`、`flag`、`jumpChapter`、`gainItem` 等

### 编辑器面板

| 面板 | 功能 |
|------|------|
| 章节编辑面板 | 选中章节节点时显示：步骤列表 + 步骤表单 |
| 结局编辑面板 | 选中结局节点时显示：结局属性 + 来源章节列表 |
| 分组编辑面板 | 选中分组矩形时显示：成员列表 + 颜色/透明度设置 |
| 树视图 | 所有节点的跳转关系可视化（SVG） |
| 导出面板 | 导出为 JS 模块或 JSON/资源包 |

### 数据同步

编辑器通过 `localStorage('galgame-editor-data')` 与主应用双向同步：

- **编辑器保存** → localStorage → 主应用检测到新数据后自动合并
- 同步数据 24 小时内有效（过期自动忽略）

### 导出功能

| 格式 | 说明 |
|------|------|
| JS 代码 (`Ctrl+Shift+E`) | 导出为 ES Module 格式，可直接替换 `game/chapters/` 文件 |
| JSON (`Ctrl+E`) | 导出为标准 JSON 文件 |
| 资源包 | 导出为 ZIP 格式的资源包（含图片资源） |

## 技术实现

编辑器在 `editor-app.js` 中实现：

- **树分析** (`analyzeTree`) — 遍历章节数据构建邻接表、计算入度
- **自动布局** — 基于入度的层次化布局算法
- **SVG 渲染** — 使用原生 SVG 元素绘制连线和节点
- **节点类型系统** — `'chapter'` / `'ending'` 统一处理
- **拖拽** — 通过鼠标事件实现节点自由拖拽（支持多选拖拽）
- **缩放/平移** — 支持画布缩放和平移（以鼠标位置为中心）
- **端口拖拽** — 底部出口端口可拖拽到目标节点建立新连接（带磁吸）
- **全局禁用选中** — CSS `user-select: none` + JS `selectstart`/`dragstart` 拦截，输入框不受影响
